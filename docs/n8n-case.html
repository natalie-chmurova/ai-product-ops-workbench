Keep a ClickUp board in sync with your product meetings

An n8n workflow that reads a meeting transcript, compares it against the live ClickUp board, and does the right thing per point: create genuinely new tasks, comment new context onto tasks that already exist, and escalate the calls it is not confident about to a human in Telegram — without ever creating a duplicate.

Stack: n8n · Claude (Anthropic) · ClickUp API · Telegram

The problem

Meetings don't just create work — they update it. A follow-up sync is mostly status changes on tasks that already exist. Two naive approaches both fail: doing it by hand is slow and gets skipped; blindly auto-creating tasks floods the board with duplicates and lets an LLM make triage calls it shouldn't make alone. The real job is reconciliation — match each point to the board, decide new vs. update, and keep a human on the doubtful ones.

Guardrails & judgment

Three deliberate choices carry this. Structured, tolerant parsing: asking the model for raw JSON broke repeatedly on quotes and line breaks, so I moved to a labelled text format plus a parser that drops malformed fragments. A grounded match: the agent only ever comments on a task id that actually exists on the board — an update pointing anywhere else is forced to a human. Confidence over bravado: the SOP tells the agent to lower its confidence rather than guess, which is exactly what routes the ambiguous calls to Telegram.

Evals

Extraction is measured, not vibes-checked: a labeled ground-truth transcript (8 must-have + 3 debatable tasks) and an LLM-as-judge harness scoring precision/recall per run. Current pipeline: 100% recall · 100% precision over 3 runs (evals/). Next: harder, messier transcripts to stress the numbers.
