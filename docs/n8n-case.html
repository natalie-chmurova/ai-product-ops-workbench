Meeting → ClickUp tasks, with an agent that knows when to ask

An n8n workflow that turns a raw product-meeting transcript into prioritized ClickUp tasks — and routes every decision it is not confident about to a human for one-tap approval in Telegram.

Stack: n8n · Claude (Anthropic) · ClickUp API · Telegram

The problem

Every product sync produces decisions, owners, and bugs — buried in a messy transcript. Turning that into a structured backlog is high-effort, low-leverage work. Full automation has the opposite failure mode: an LLM will happily make triage calls it should not be making alone. The interesting problem is the middle: automate the confident calls, escalate the doubtful ones.

Guardrails & judgment

The first version asked the model for raw JSON — and it broke repeatedly on quotes and line breaks. I switched to a labelled text format the model produces reliably, plus a parser that tolerates format drift and drops malformed fragments. The agent's SOP explicitly instructs it to lower its confidence rather than guess when information is missing — which is exactly what routes work to the human. Structured output, tolerant parsing, and escalation rules matter more than the happy path.

Evals

Extraction is measured, not vibes-checked: a labeled ground-truth transcript (8 must-have + 3 debatable tasks) and an LLM-as-judge harness scoring precision/recall per run. Current pipeline: 100% recall · 100% precision over 3 runs on the benchmark transcript (evals/). Next: harder, messier transcripts to stress the numbers.
