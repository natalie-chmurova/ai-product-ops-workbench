n8n · agentic automation · portfolio case

Meeting → ClickUp tasks

An n8n workflow that turns a raw product-meeting transcript into prioritized, ClickUp-ready tasks — automatically, with no manual triage.

Stack: n8n · Claude (Anthropic) · ClickUp API

The problem

Every product sync produces decisions, owners, and bugs — buried in a messy transcript. Turning that into a structured, prioritized backlog is high-effort, low-leverage work that gets done late, inconsistently, or not at all. Decisions get lost; bugs slip through.

Guardrails & judgment

The first version asked the model for raw JSON — and it broke repeatedly on quotes and line breaks inside long descriptions. I switched to a labelled text format the model produces reliably, plus a parser that tolerates format drift and drops malformed fragments. That is the real lesson of building agentic flows: structured output, tolerant parsing, and exception handling matter more than the happy path.
